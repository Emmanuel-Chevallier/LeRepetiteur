Nous avons mis en place la partie "Enseignant" de l'application InterroPedago. Pour finaliser le déploiement de la partie "Étudiant", j'ai besoin que le conteneur Docker accède aux données que je synchronise.

(Ce dossier contient les sous-dossiers `courses`, `students`, `global`).

2. Configuration Requise (Docker)
---------------------------------
L'application Étudiant tourne (ou tournera) dans un conteneur Docker. Elle s'attend à trouver ces données dans `/app/data`.

Pour que la mise à jour soit automatique, l'idéal serait de monter directement mon dossier de "staging" comme volume de production.

Option A : Montage direct (Recommandé si Docker est sur la même machine/FS)
Dans le fichier docker-compose.yml de l'application étudiant :
------------------------------------------------------------
services:
  student-app:
    # ...
    volumes:
      - /space/home/chevallier/InterroPedago/data:/app/data
    # ...
------------------------------------------------------------
Cela permet à l'application de voire immédiatement les fichiers que je déploie via SFTP.

Option B : Script de Synchro (Si partitionnement strict)
Si le Docker ne peut pas accéder à /home/chevallier pour des raisons de droits, un script cron simple serait nécessaire (ex: rsync) pour copier périodiquement le contenu de `/space/home/chevallier/InterroPedago/data` vers le volume de production dédié (ex: `/opt/interropedago/data`).

Merci d'avance pour ton aide !
