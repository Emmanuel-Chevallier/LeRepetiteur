1. Architecture Globale

InterroPedago V6 repose sur une architecture moderne de type client-serveur.

  • Backend : Développé en Python avec FastAPI pour exposer des API RESTful rapides et asynchrones.
  • Frontend : Composé de HTML/CSS/JS natif, sans framework lourd, garantissant une exécution rapide. Le rendu du LaTeX est assuré par MathJax.
  • Stockage de Données : Basé sur des fichiers JSON structurés dans le répertoire data/, permettant une portabilité maximale et un déploiement facile sans base de données externe.
  • Moteur IA : L'ensemble de l'intelligence artificielle (génération, correction, chatbot) s'appuie sur les modèles de langage Google Gemini (gemini-3.1-pro-preview).

2. Gestion des Étudiants

Le système assure l'anonymat et la gestion simplifiée des cohortes.

Fonctionnalités

  • Anonymisation : Chaque étudiant est identifié en interne par un code unique à 4 chiffres (ex: 1234). Sur les copies, seul ce numéro ("Candidate #1234") apparaît pour garantir un traitement impartial.
  • Importation CSV : Permet d'importer une liste d'étudiants (Nom, Prénom). Le système gère automatiquement les homonymes en ajoutant un suffixe (ex: "Dupont (2)").
  • Génération de mots de passe : Un clic permet de générer des mots de passe aléatoires de 6 caractères alphanumériques pour tous les étudiants n'en possédant pas.
  • Génération par lots : Utile pour des tests ou des inscriptions de masse, crée N comptes fictifs numérotés consécutivement.

3. Gestion des Cours & Configuration

La gestion des supports de cours est au cœur de la génération de quizzes.

  • Source LaTeX : L'enseignant téléverse le fichier source .tex de son cours complet.
  • Fenêtrage contextuel : Plutôt que de fournir l'intégralité du cours à l'IA pour chaque génération, l'enseignant définit un intervalle de lignes (Début / Fin). Cela permet de cibler précisément le dernier chapitre abordé en classe pour les évaluations continues.
  • Prévisualisation : Un outil intégré permet d'extraire et d'afficher le code LaTeX de l'intervalle sélectionné pour vérifier la portée du sujet.

4. Génération de la Banque de Questions

La V6 introduit le concept de "Banque de Questions" pour plus de contrôle pédagogique.

Implémentation

À partir de l'intervalle de cours sélectionné (cf. Section 3), l'enseignant demande la génération de N questions.

  1. Le Backend envoie le segment LaTeX et le Prompt de la Banque à Gemini.
  2. Gemini retourne une liste brute d'items LaTeX.
  3. Le Backend parse ces questions et les affiche.
  4. Boucle de Feedback : L'enseignant peut fournir des instructions supplémentaires dans un champ de commentaire. Le système renvoie alors à l'IA les questions actuelles + le commentaire + le cours, afin de raffiner les questions.
  5. Les questions finalisées cochées sont enregistrées et formeront le pool ("nouvelles questions") pour les quizzes.

5. Génération Personnalisée des Quizzes

C'est l'atout majeur d'InterroPedago : des évaluations différenciées basées sur les lacunes individuelles.

Algorithme de Génération

Lorsqu'un quiz est généré pour un étudiant, l'IA reçoit un prompt contenant :

  • Son Historique : La liste des questions posées lors des précédents quizzes de ce cours et les notes/feedbacks obtenus.
  • La Banque Sélectionnée : Les questions approuvées à l'étape précédente.
  • Le Cours : Le segment LaTeX de référence.

La directive par défaut impose de piocher :
- n_new questions depuis la banque sélectionnée (évaluant le nouveau cours).
- n_old questions depuis l'historique de l'étudiant parmi celles qu'il a échouées (principe de répétition espacée et de remédiation).

Modes de génération

  • Prochain Quiz (Tous) : Itère le compteur de quiz et génère un sujet PDF pour tous les étudiants inscrits au cours. À la fin, tous les PDFs sont concaténés en un seul document global (facile à imprimer).
  • Régénérer (Tous) : Écrase le dernier quiz généré si on souhaite changer un paramètre global.
  • Individuel : Permet de cibler un étudiant précis et un numéro de quiz spécifique pour re-générer son sujet indépendamment du reste de la classe.

6. Correction Automatique (Smart Grading)

La correction est entièrement automatisée grâce à l'analyse multimodale de Gemini.

Processus (V2)

  1. Upload Massif : L'enseignant scanne toutes les copies en un seul PDF (ou images). L'interface permet de diviser le PDF (ex: 2 pages par copie).
  2. Identification (OCR) : Le système analyse l'en-tête de chaque copie pour identifier l'ID de l'étudiant, le cours, et le numéro de quiz. L'ID (Candidate #1234) est croisé avec la base de données.
  3. Correction Contextuelle : Pour chaque copie identifiée, le système récupère en base le sujet précis qui avait été généré pour cet étudiant.
  4. Évaluation IA : L'image de la copie étudiante ET le fichier PDF du sujet original sont envoyés à Gemini. Le modèle est instruit de lire les réponses manuscrites, de les comparer aux questions du sujet, d'attribuer une note pour chaque question, et de formuler un feedback pédagogique bilingue.
  5. Le résultat est structuré en JSON et enregistré dans le dossier de l'étudiant.

7. Carnet de Notes, Présence & Synthèse

  • Carnet de Notes : Extrait dynamiquement les données JSON de correction pour générer un tableau croisé Étudiants / Quizzes. Il inclut un graphique d'évolution (moyenne et écart-type) tracé avec Plotly. Les notes automatiques peuvent être modifiées manuellement (elles apparaissent alors en jaune).
  • Présence : Système classique d'appel avec date et heure, stocké par cours.
  • Synthèse Globale : Après une session d'évaluation, le système peut lire toutes les corrections d'un quiz et faire appel à l'IA pour générer un Rapport Markdown. Ce rapport résume la moyenne globale et identifie les points du cours les moins bien compris par l'ensemble de la classe, guidant l'enseignant sur les points à revoir.

8. Espace Étudiant & Chatbot Tutoriel

L'étudiant dispose de sa propre interface sur le portail student_server.

  • Tableau de Bord : L'étudiant visualise ses cours et la liste de ses quizzes passés.
  • Correction Détaillée : Pour chaque quiz, l'étudiant accède au scan original de sa copie, au sujet généré pour lui (PDF), et au tableau de correction détaillé (note et feedback du correcteur IA par question).
  • Chatbot IA Pédagogique : Un assistant virtuel est intégré. Ce n'est pas un LLM générique : il a accès en temps réel à l'historique complet des évaluations de l'étudiant. Si l'étudiant pose une question sur le cours, le Chatbot peut y répondre en faisant le lien avec une erreur que l'étudiant a commise dans un de ses quizzes précédents, offrant un accompagnement extrêmement ciblé.

9. Guide Détaillé de l'Interface (Dictionnaire des Boutons)

Ce niveau de description est destiné aux utilisateurs finaux (enseignants) pour comprendre le rôle précis de chaque élément interactif du tableau de bord.

En-tête & Vues globales

  • 🚀 Déployer ▼ : Ouvre un menu déroulant permettant d'exporter les fichiers du backend vers le serveur distant de l'université (SFTP) ou de lancer des tests locaux.
  • 👁️ Switch View : Permet d'ouvrir rapidement l'interface Étudiant dans un nouvel onglet pour vérifier ce que voient les apprenants.
  • 📖 Documentation : Ouvre la présente page d'aide dans un nouvel onglet.

Section 0. Gestion des Étudiants

  • Ajouter Étudiant : Crée manuellement un profil étudiant en entrant un identifiant à 4 chiffres.
  • 📁 Importer CSV : Ouvre une fenêtre système pour sélectionner un fichier CSV. Le système lira les colonnes "Nom" et "Prénom" pour peupler la base de données.
  • 🔑 Générer les Accès : Attribue un mot de passe alphanumérique unique et sécurisé à chaque étudiant qui n'en possède pas encore.
  • Générer N Étudiants : Outil de test/initialisation rapide pour créer une série de profils génériques (ex: de 2000 à 2030).
  • 👁️ Montrer/Cacher les emails : Affiche ou masque la colonne des adresses email dans le tableau principal pour des raisons de confidentialité.
  • 👁️ Cours affichés ▼ : Menu déroulant pour sélectionner quelles colonnes de notes (de quels cours) afficher dans le grand tableau récapitulatif.
  • ☑ Tout / 📋 Copier : Permet de sélectionner rapidement tous les étudiants et de copier leurs adresses email dans le presse-papier pour un envoi groupé.
  • Inscrire / Désinscrire : Boutons de couleur (Vert/Rouge) situés dans le tableau. Permettent d'ajouter ou retirer un étudiant du cours sélectionné.
  • ✏️ (Éditer) : Ouvre une fenêtre modale pour corriger le nom, prénom ou l'email d'un étudiant.
  • 🗑️ (Supprimer) : Supprime définitivement l'étudiant et toutes ses données associées (copies, notes).
  • 👁️ Voir Historique : Ouvre une fenêtre détaillée affichant les conversations Chatbot et les copies de l'étudiant.

Section 1. Gestion des Cours

  • Créer : Initialise un nouveau cours dans la base de données avec le titre fourni.

Configuration du Cours (Sections 1 & 2)

  • Upload : Téléverse le fichier source .tex contenant le contenu pédagogique du cours sélectionné.
  • 👁️ Voir : Affiche le code LaTeX complet du cours dans un encadré redimensionnable pour lecture.
  • 👁️ Prévisualiser : Affiche uniquement la portion du code LaTeX correspondant à l'intervalle défini par les champs "Ligne Début" et "Ligne Fin".

Section 3. Banque de Questions

  • Valider : Lance l'appel à l'IA (Gemini) pour générer le nombre de questions spécifié, en se basant sur l'intervalle de cours et le prompt défini.
  • 🔄 Soumettre les commentaires : Relaie vos remarques textuelles à l'IA pour qu'elle corrige, reformule ou adapte les questions qui viennent d'être générées.

Section 4. Génération des Quizzes

  • 🆕 Prochain Quiz (tous) : Mode de génération standard. Incrémente le numéro de quiz et prépare les sujets pour toute la classe.
  • ♻️ Régénérer dernier (tous) : Mode de correction. Écrase le dernier quiz généré pour la classe entière (utile si vous avez modifié le prompt à la dernière minute).
  • 👤 Individuel : Mode de rattrapage. Affiche des menus déroulants pour sélectionner un étudiant spécifique et regénérer son sujet indépendamment.
  • 💾 Sauvegarder (Prompt) : Enregistre définitivement les modifications apportées au texte du prompt pour ce cours.
  • ↺ Restaurer défaut : Réinitialise la zone de texte avec le prompt originel programmé par les développeurs.
  • 🚀 Lancer la Génération : Déclenche le processus asynchrone : interroge l'IA pour chaque étudiant, compile les fichiers LaTeX individuels, puis génère le PDF global de la classe.

Section 5. Carnet de Notes & Présence

  • Rafraîchir : Force le rechargement du tableau des notes depuis les fichiers JSON, utile si une correction en arrière-plan vient de se terminer.
  • Ajouter une séance : Insère une nouvelle colonne dans le tableau d'appel à la date et l'heure spécifiées.

Section Correction Automatique (Smart Grading)

  • 📁 Sélectionner des fichiers : Ouvre l'explorateur de fichiers pour sélectionner vos scans (PDF ou Images). Le système divisera automatiquement les PDFs multi-pages selon le réglage "Pages par copie".
  • 🚀 Lancer la Correction : Envoie les copies en attente à l'IA pour lecture OCR, évaluation, et génération de feedback. Le bouton indique le nombre de copies prêtes (X).
  • Voir / Voir Questionnaire : Permet de prévisualiser l'image scannée ou le PDF du sujet original avant de lancer la correction.
  • X (Rouge) : Retire la copie de la file d'attente sans la corriger.

Section Synthèse Globale

  • 📝 Modifier le Prompt de Synthèse : Ouvre le champ pour ajuster les instructions données à l'IA concernant la rédaction du rapport de classe.
  • Générer la Synthèse : Analyse toutes les corrections du quiz sélectionné et produit un rapport Markdown identifiant les difficultés principales rencontrées par les étudiants.

Autres Boutons

  • 🗑️ Supprimer ce Cours : Bouton de danger (rouge) situé en bas. Supprime de manière irréversible le cours et tous les fichiers associés (sujets, scans, notes).
